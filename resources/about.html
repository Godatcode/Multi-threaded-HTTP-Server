Project Overview

This project implements a production-grade HTTP/1.1 server from scratch using low-level socket programming. It demonstrates deep understanding of network protocols, concurrent programming, and security best practices.

üèóÔ∏è Architecture

The server uses a thread pool pattern for efficient concurrency management. When a client connects:

  1. The main thread accepts the connection
  2. Connection is placed in a thread-safe queue
  3. An available worker thread picks up the connection
  4. The worker handles all requests on that connection
  5. Connection is closed or kept alive based on headers

üîß Technical Implementation

Socket Programming

Uses Python's socket module to create TCP sockets. The server binds to a specified host and port, listens for incoming connections with a queue size of 50, and manages socket lifecycle for both persistent and non-persistent connections.

# Socket creation and binding socket = socket.socket(socket.AF_INET, socket.SOCK_STREAM) socket.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1) socket.bind((host, port)) socket.listen(50)

Thread Pool Management

Implements a configurable thread pool with:

  • Thread-safe connection queue using mutex locks
  • Configurable pool size (default: 10 threads)
  • Automatic connection queuing when all threads are busy
  • Graceful shutdown mechanism
  • Real-time active thread counting

HTTP Protocol Implementation

Fully compliant with HTTP/1.1 specification (RFC 7230-7235):

  • Request Parsing: Extracts method, path, version, headers, and body
  • Response Building: Proper status lines, headers, and body encoding
  • Content-Type Handling: Correct MIME types for all file types
  • Content-Length: Automatic calculation for all responses
  • Date Headers: RFC 7231 compliant timestamp formatting

Binary File Transfer

Special implementation for binary file downloads:

  • Files opened in binary mode ('rb')
  • Content-Type: application/octet-stream
  • Content-Disposition header triggers download
  • 4KB buffer size for efficient streaming
  • Checksum verification support
# Binary file serving with open(file_path, 'rb') as f: content = f.read() headers = { 'Content-Type': 'application/octet-stream', 'Content-Disposition': f'attachment; filename="{filename}"', 'Content-Length': str(len(content)) }

üîí Security Features

Path Traversal Protection

Implements multiple layers of protection:

  • Blocks requests containing .. or //
  • Canonicalizes paths using os.path.abspath()
  • Ensures resolved path stays within resources/ directory
  • Returns 403 Forbidden for unauthorized access attempts
  • Logs all security violations for monitoring

Host Header Validation

Validates the Host header against server configuration:

  • Returns 400 Bad Request if Host header is missing
  • Returns 403 Forbidden if Host doesn't match server address
  • Supports multiple valid formats (localhost, 127.0.0.1, with/without port)

‚ö° Connection Management

HTTP Keep-Alive

Supports persistent connections to reduce overhead:

  • HTTP/1.1 default: keep-alive enabled
  • HTTP/1.0 default: connection closed after response
  • Respects Connection header from client
  • 30-second timeout for idle connections
  • Maximum 100 requests per connection
# Keep-Alive response headers Connection: keep-alive Keep-Alive: timeout=30, max=100
